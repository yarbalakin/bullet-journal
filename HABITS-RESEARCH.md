## 1. Наука о формировании привычек

### Петля привычки (Habit Loop)
Нейробиологи MIT: **cue → routine → reward**. Дофамин выделяется сильнее в момент *ожидания* награды, чем при её получении — отсюда аддиктивность трекеров.

Как должно работать в приложении:
- **Cue**: напоминание в момент якорного ритуала (утренний кофе, открытие телефона)
- **Routine**: одно касание для отметки — никаких модальных окон
- **Reward**: анимация + звук + цвет **в момент отметки**, а не потом

### Don't Break the Chain (Стратегия Сейнфелда)
Джерри Сейнфелд ставил крест на бумажном календаре каждый день, когда писал шутки. Психологические двигатели:
- **Loss aversion**: Канеман и Тверски — потеря ощущается вдвое сильнее, чем равноценное приобретение. Серия в 20 дней субъективно огромна.
- Duolingo: разделение дневной цели и стрика → +10.5% пользователей на стрике за 20 дней
- Streak Freeze: увеличила долгосрочный retention на 23% (ACM CHI 2019)

### Implementation Intentions
Голлвитцер (NYU, 1999): **"если [ситуация Y], то я сделаю [Z]"**. Метаанализ: цели с implementation intentions выполняются в 2-3 раза чаще.

В приложении: при создании привычки обязательный вопрос "когда, где, после чего?". Напоминание приходит не "в 8:00", а как привязка к якорному событию.

### Habit Stacking
BJ Fogg (Stanford): новое поведение паразитирует на уже существующем нейронном пути. "После [якорного момента] я сделаю [новое поведение]". 

Для приложения: группировать привычки в рутины (утренняя / рабочая / вечерняя), а не разрозненный список.

### Оптимальное количество привычек
- Лалли и коллеги (UCL, 2010): среднее время формирования привычки — **66 дней** (диапазон 18–254)
- Ego depletion (Баумайстер): самоконтроль — исчерпаемый ресурс
- Рекомендация: **1-3 активных привычки** одновременно на стадии формирования
- Разблокировка новых привычек после 30+ дней предыдущей

### Работа с пропусками без разрушения мотивации
"What-the-hell effect" (Поливи): один срыв ведёт к полному отказу. Исследование JPSPsych (2020): пользователи streak-приложений на 63% чаще бросали после первого пропуска.

**Never Miss Twice** (James Clear): "Пропустить один раз — случайность, пропустить дважды — начало новой привычки."

**Penn Reset Rule**: пользователи с "вернуться в течение 24 часов" на 71% чаще сохраняли привычку долгосрочно.

### Социальная подотчётность
ASTD: вероятность достижения цели — 65% с партнёром, 95% при назначенных встречах. Самый мощный одиночный фактор.

Градация силы воздействия (от слабого к сильному):
1. Публичный профиль / лента
2. Уведомление другу о прогрессе
3. Совместный стрик с партнёром
4. Финансовый залог (Beeminder)
5. Прямые последствия для других (Habitica-модель)

## 2. Лучшие приложения — ключевые механики

### Streaks (iOS, Apple Design Award 2016)

**Главная идея**: to-do лист, который помогает формировать привычки. Максимум 24 задачи — сознательное ограничение.

**Ключевые фичи:**
- **Прогресс-кольца** — каждая привычка = круг, заполняется по выполнению (как Apple Activity Rings)
- **Тройная сенсорная обратная связь**: анимация + звук + тактильная вибрация при выполнении
- **Long-press для завершения** — защита от случайных кликов, создаёт осознанность
- **3 типа задач**: да/нет, негативные (засчитывается если НЕ сделал), с таймером
- **Гибкое расписание**: конкретные дни, N раз в неделю
- **Паттерны по дням недели** в статистике — в какой день чаще выполняется
- **Паттерны по времени суток** — когда обычно закрывается задача
- **Best Streak** — лучшая серия за всё время как долгосрочный мотиватор
- Apple Health интеграция — автозакрытие задач по данным шагомера
- 78 цветовых тем, 600+ иконок

**Философия**: loss aversion через один мощный якорь — стрик. Минимум геймификации намеренно.

---

### Habitica (Web/iOS/Android, open source)

**Главная идея**: реальная жизнь = RPG-игра. Каждое выполнение = XP + золото. Провал = урон HP.

**Ключевые фичи:**
- **Три типа задач**: Habits (любое время), Dailies (с расписанием, провал = урон), To-Dos (разовые)
- **Цветовая система задач (Task Value)**: цвет меняется по истории выполнения — жёлтый (нейтральный) → зелёный/синий (часто выполняется) → оранжевый/красный (часто пропускается). Визуальная аналитика без цифр.
- **Custom Rewards**: "заработай 50 золота → посмотри серию сериала". Пользователь сам определяет награды.
- **Классы персонажа** (разблокируются на уровне 10): Warrior, Mage, Healer, Rogue — разные бонусы.
- **Party + квесты**: провал Daily = босс атакует **всю группу**. Сильнейший механизм социального давления.
- **Пауза урона**: отдых без штрафов, честно перед собой.
- **Чеклист внутри задачи**: подзадачи внутри одной привычки.
- **Дроп-система**: случайные яйца, зелья, питомцы при выполнении — переменное вознаграждение.
- Группы: +51% social engagement, тематические челленджи: +39% retention.

**Философия**: переменное подкрепление + социальное давление. Для людей с внешней мотивацией.

### Fabulous (iOS/Android, из Duke Behavioral Economics Lab)

**Главная идея**: не трекер, а структурированный коучинг. Интерфейс = нарративный контейнер для самодисциплины.

**Ключевые фичи:**
- **Journeys**: многодневные программы как "главы истории": "In which [имя] learns how to manufacture a great night's sleep"
- **Habit Stacking**: один стартовый микроритуал (стакан воды) + постепенное наслоение
- **Routine Builder**: рутина как связанная цепочка, а не разрозненный список
- **Pre-commitment contracts**: подпись отпечатком пальца перед работой
- **Anchor sounds + таймеры**: звук капли воды для гидратации — ассоциированный триггер
- **Full-screen mode** при выполнении: иллюстрация + таймер + фоновая музыка = "ритуальное пространство"
- **31-экранный onboarding**: sunk cost через инвестицию времени → эмоциональная привязанность до начала работы
- **Three Mountains**: три фазы роста (Фундамент → Мышление → Видение) — долгосрочная карта

**Философия**: "ты пишешь следующую главу своей истории". Для людей с амбициями и внутренней мотивацией.

### Finch (iOS/Android — self-care pet)

**Главная идея**: забота о виртуальной птице как прокси для заботы о себе. Снижает тревогу перфекционизма.

**Ключевые фичи:**
- **Виртуальный питомец**: растёт через жизненные стадии по мере выполнения задач
- **Care Transference**: "я делаю это для птицы" — снижает сопротивление
- **Adventures**: питомец отправляется в 6-часовые приключения по накоплении Energy → органичные точки возврата без навязчивых уведомлений
- **Zero-shame mechanics**: никаких штрафов за пропуск, птица терпеливо ждёт
- **One Metric**: Energy — один простой показатель дня, не множество конкурирующих метрик
- **Home Screen Widget**: живой питомец на рабочем столе — постоянное ненавязчивое напоминание
- **Social без конкуренции**: визиты друзей, подарки без leaderboards
- **Seasonal events**: ежемесячное обновление визуального оформления

**Для ADHD**: "единственное приложение, которое выжило у меня дольше двух недель."

### Strides (iOS — goal-first трекер)

**Главная идея**: "am I on track to hit my goal?" — ориентация на результат.

**4 типа трекеров (архитектурно разные сущности):**

1. **Habit Tracker** (да/нет, свайп): хорошие и плохие привычки
2. **Target Tracker** (числовая цель к дате): "прочитать 24 книги к 31 декабря" — автоматический расчёт необходимого темпа
3. **Average Tracker** (скользящее среднее): средний сон за неделю, среднее давление
4. **Project/Milestone Tracker**: разбивка сложной цели на вехи с процентом выполнения

**Ключевые фичи:**
- **Pace Line**: линия "где ты должен быть сейчас". Зелёный = на треке, красный = отставание.
- **Projected Line**: экстраполяция текущего темпа в будущее
- **Bad Habit режим**: прогресс-бар начинается заполненным (100%) и убывает — противоположный психологический паттерн
- **Tags + Reports по тегу**: группировка по сфере жизни (здоровье, финансы), агрегированная статистика
- **Retroactive logging**: тап на прошедшую дату в календаре
- 150+ готовых шаблонов, 15 звуков уведомлений
- Due-aware напоминания: не срабатывают если привычка уже отмечена

### HabitBull (iOS/Android — consistency-first)

**Главная идея**: "don't break the chain" — ориентация на процесс.

**Ключевые фичи:**
- **Трёхцветная dot-система** (ключевой механизм):
  - Зелёная точка: день обязателен + выполнен → серия продолжается
  - Жёлтая точка: день необязателен → серия продолжается (не штрафует за нерабочие дни)
  - Красная точка: день обязателен + провален → серия прерывается
- **HabitBull Score**: составная метрика (серии + успешность), таргет 100% — не просто процент
- **Habit Notes**: long tap на день = заметка. В Monthly View все заметки за месяц рядом с календарём.
- **Повторяющиеся напоминания в промежутке**: "каждые 2 часа с 9 до 21" — для привычек типа "пить воду"
- Monthly dot-calendar как основная вьюха — интуитивно читаемая
- CSV экспорт
- Community-форум внутри категории привычки

---

### Loop Habit Tracker (Android, open source)

**Главная идея**: habit score вместо стрика — экспоненциальное сглаживание.

**Ключевые фичи:**
- **Habit Score (0-100%)**: несколько пропусков после долгой серии НЕ обнуляют прогресс. Ежедневная привычка достигает 80% силы за месяц, 99% за три.
- **Skip ≠ Miss**: долгое нажатие = "пропуск уважительный" — не влияет на счёт
- **5 графиков** на экране привычки (переключаются свайпом): линейный (habit score), столбчатый (количество), heatmap, серии, частота
- **Числовые привычки**: порог "не менее X" или "не более X" с единицей измерения
- Два типа: бинарные и измеримые
- Гибкое расписание (N раз/нед, раз в X дней)
- Нет аккаунта, нет облака, нет рекламы, нет пэйволла

### Notion как трекер (гибкость > специализация)

**Ключевые паттерны:**
- **Day Score**: процент выполненных привычек за день как агрегат
- **Progress Bar**: строковая формула ████░░░ (80%) — прямо в базе данных
- **Motivational messages**: условные строки по уровню выполнения (>80% = "отличный день", <40% = "ничего, завтра лучше")
- **Repeating Templates**: запись дня создаётся автоматически — не нужно помнить
- **Связь с целями**: показывает "зачем" рядом с "что" — привычки в контексте проектов и целей
- **Галерея / карточка дня**: объединяет трекер и журнал в одном месте
- **Heatmap шаблоны**: 53-недельный GitHub-style с 6-уровневым градиентом

## 3. UI/UX паттерны и визуализация

### Тепловые карты (Heatmap Calendar)

**GitHub-стиль** — самый распространённый и читаемый:
- Сетка 52×7 (год), интенсивность цвета = активность
- Ячейки строго квадратные
- Видны провалы, полосы активности, сезонные паттерны
- Минимум активности = светло-серый, максимум = насыщенный основной цвет

**Duolingo-стиль** — тепловая карта как нарратив:
- Glow-эффект на активных ячейках
- Анимированное пламя рядом с горящей серией
- Пользователи с серией 7+ дней возвращаются в 2.4 раза чаще

### Счётчики серий (Streak Counters)

**Психология:**
- Sunk Cost: 200-дневная серия = сотни дней инвестиций. Боль от потери сильнее радости от продолжения.
- Goal-Gradient Effect: чем ближе к рекорду, тем выше мотивация.
- Streak Freeze: +14% удержание на 14-й день (Duolingo данные).

**Визуальные решения:**
- Большое жирное число + иконка огня/звезды
- Анимация при milestone (7, 14, 30, 100, 365 дней) — отдельная анимация, не просто смена числа
- Цвет иконки по накалу серии: жёлтый → оранжевый → красный
- "Сегодня" выделен: видно, есть ли уже чекмарк

### Habit Stacking / Рутинный интерфейс

**Лучший паттерн — разбивка по времени суток:**
- 3 секции: Утро / День / Вечер (иконки рассвет/солнце/луна)
- Прогресс-бар секции: "Утро: 3/5 выполнено" — мотивирует закрыть блок целиком
- Показывать только актуальные привычки прямо сейчас

### Кольца и круговые индикаторы (Activity Rings)

**Apple Watch — эталон:**
- Принцип замкнутости Гештальта: незакрытое кольцо создаёт "ментальный зуд"
- 24-часовой сброс: дедлайн близко, но не давит как долгосрочная цель

**Веб-адаптация:**
- Одно кольцо на привычку — заполнение по дням недели (7 сегментов)
- Центр кольца: число выполнений или процент
- При закрытии — анимация "взрыва"
- Уникальный цвет кольца на каждую привычку

**Прогресс-бар** работает лучше для количественных целей (10 000 шагов, 2 литра воды):
- +30% к соблюдению привычки при наличии мгновенной визуальной обратной связи (исследования)

### Недельный и месячный обзор

**Недельный вид:**
- 7 ячеек в строку, каждая строка = одна привычка
- Хорошо работает при 5-8 привычках
- Цветное кодирование: различаемые без чтения текста

**Месячный вид:**
- Мини-сетка 4-5 недель, каждый день = точка/квадрат
- Для инсайтов: "в середине месяца всегда провал"

**Паттерн "аналитика отдельно от действия":**
- Интерфейс выполнения (сегодня, быстрое касание) и интерфейс анализа (графики, тренды) — разделять. Смешение убивает оба.

### Цветовое кодирование

**Три уровня:**
- **Базовый (бинарный)**: выполнено = насыщенный цвет, пропущено = серый/пустой
- **Средний (градиентный)**: частичное выполнение = промежуточный оттенок
- **Продвинутый (mood + completion)**: оттенок = настроение, интенсивность = выполнение привычек. Видна корреляция "когда хорошо делаю = хорошо себя чувствую"

**Палитра:**
- Тёмный фон: #121212 или #1E1E1E + слабо насыщенные акценты
- 8-12 предустановленных цветов на выбор пользователя

### Ключевые принципы вовлечённости (данные)

| Механика | Эффект |
|---|---|
| Серия 7 дней | +140% вероятность вернуться завтра (Duolingo) |
| Анимация при закрытии кольца | +1.7% удержание на 7-й день |
| Streak Wager | +14% удержание на 14-й день |
| Прогресс-бар + мгновенный цвет | +30% соблюдение привычки |
| Streak Freeze | Снижает отток при пропуске дня |

**7 принципов UI:**
1. **Одно касание = действие.** Отметить выполненной — максимум 1 тап, без модалок.
2. **Мгновенный отклик.** Цвет меняется без задержки.
3. **Прогресс, а не перфекционизм.** Пропуск = не конец.
4. **Вариативные уведомления.** Один текст умирает через 2-4 недели.
5. **Пустой экран = враг.** 3-5 стартовых привычек при онбординге сразу.
6. **Микроанимации на milestone.** 7, 30, 100 дней — отдельная анимация.
7. **Социальное сравнение** по % выполнения (не по дням) — уравнивает новичков и ветеранов.

## 4. Большой список фич для реализации

### Must-Have (максимальная доказательная база)

- [ ] **Два типа привычек**: да/нет и числовые (с порогом "не менее / не более X")
- [ ] **Habit Score** вместо простого стрика: экспоненциальное сглаживание (алгоритм Loop). Пропуск не обнуляет — ослабляет.
- [ ] **Трёхцветная dot-система** (зелёный/жёлтый/красный по аналогии HabitBull) — обрабатывает необязательные дни без потери серии
- [ ] **Skip-статус**: "уважительный пропуск" — не влияет на счёт
- [ ] **Отметка в одно касание** с главного экрана, без перехода в детали
- [ ] **Celebration animation** прямо в момент отметки (петля привычки)
- [ ] **Streak + Best Streak** — отдельный показатель лучшей серии за всё время
- [ ] **Never Miss Twice** как коммуникация: нейтральное, не обвинительное сообщение при пропуске
- [ ] **Habit Notes**: long-tap на день = добавить заметку к этому дню
- [ ] **Прогресс-бар утреннего блока**: "Утро: 3/5 выполнено"
- [ ] **Day Score**: процент выполненных привычек за день как агрегат в верхней части

### High-Value (хорошая доказательная база)

- [ ] **Разбивка по времени суток**: Утро / День / Вечер с отдельными секциями
- [ ] **Негативные привычки**: прогресс-бар начинается полным и убывает (не сделал = успех)
- [ ] **Pace Line** для числовых привычек: линия "где ты должен быть сейчас" при цели к дате
- [ ] **Projected Trend Line**: экстраполяция текущего темпа
- [ ] **Streak Freeze**: 1-2 "заморозки" в запасе
- [ ] **Гибкий стрик**: "N из 7 дней = серия продолжается"
- [ ] **Ретроспективный процент**: "80% за последние 30 дней" отдельно от стрика
- [ ] **Retroactive logging**: тап на прошедшую дату для редактирования
- [ ] **Tags / категории**: группировка по сфере жизни (здоровье, работа, отношения)
- [ ] **Milestone анимации**: 7, 14, 30, 100, 365 дней — отдельная анимация, не просто смена числа
- [ ] **Motivational messages**: условный текст по уровню дня (>80% = одно, <40% = другое)
- [ ] **Цветовая система задачи по истории** (Habitica-паттерн): цвет меняется автоматически по накопленной истории
- [ ] **Habit Stacking visual**: показывать цепочку привычек в рутине как связанные действия
- [ ] **Подзадачи внутри привычки** (чеклист)
- [ ] **Custom Rewards**: заработать очки → разблокировать личную награду (кофе, серия, прогулка)

### Advanced (инновационный край)

- [ ] **Target Tracker** (цель к дате): "прочитать 24 книги к 31 декабря" — автоматический расчёт темпа
- [ ] **Average Tracker**: скользящее среднее без жёсткой ежедневной цели
- [ ] **Implementation intentions**: при создании привычки вопрос "когда, где, после чего?"

- [ ] **Паттерны по дням недели**: в какой день чаще всего выполняется привычка
- [ ] **Паттерны по времени суток**: когда обычно закрывается задача

## 5. Рекомендуемая архитектура блока

### Модель данных привычки

```javascript
{
  id: string,
  name: string,
  type: 'boolean' | 'number' | 'negative',  // три типа
  schedule: {
    days: number[],     // 0-6, дни недели
    frequency: number,  // сколько раз в неделю (для "N из 7")
  },
  goal?: {
    target: number,
    deadline: string,   // ISO date
    unit: string,
  },
  category: string,     // утро / день / вечер / здоровье / работа...
  color: string,
  reminder?: { time: string, days: number[] },
  logs: {
    [date: string]: {
      status: 'done' | 'skip' | 'miss' | 'partial',
      value?: number,
      note?: string,
    }
  }
}
```

### Алгоритм Habit Score (Loop)

```javascript
// Экспоненциальное сглаживание
// alpha: сглаживающий коэффициент (Loop использует ~0.1)
function habitScore(logs, alpha = 0.1) {
  let score = 0;
  const sortedDates = Object.keys(logs).sort();
  for (const date of sortedDates) {
    const done = logs[date].status === 'done' ? 1 : 0;
    score = score + alpha * (done - score);
  }
  return Math.round(score * 100); // 0-100%
}
```

### Сильная связка для первой версии

**Минимализм (одно касание)** + **GitHub-heatmap (годовой вид)** + **Кольца (недельный вид)** + **Серия с заморозкой (удержание)** + **Разбивка утро/день/вечер** + **Day Score как агрегат** = покрывает все основные психологические механики без перегруза.

### Что НЕ делать

- Не смешивать интерфейс выполнения и аналитики на одном экране
- Не делать жёсткий стрик без grace period / freeze
- Не показывать больше 8 привычек одновременно без фильтрации
- Не использовать обвинительный тон при пропусках
- Не давать отмечать привычку через модальное окно — только inline
- Не делать геймификацию ради геймификации — только если подкреплена механикой удержания
