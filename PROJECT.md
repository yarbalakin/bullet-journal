## 1. Суть продукта

Мобильное приложение в стиле Bullet Journal — ежемесячный планировщик с визуальной обложкой, календарём, трекером настроения и трекером косметики (срок годности по штрихкоду).

**Уникальная комбинация:** BuJo-философия (migration, future log, collections) + трекер косметики + mood tracker + стикеры — в одном приложении. Аналогов нет.

**Не является:** социальной сетью, заменой Google Calendar, профессиональным органайзером.

## 3. Ключевые функции

### MVP-0: Core (быстрый запуск)

#### 3.1 Обложка месяца
- Каждый месяц — своя обложка (страница журнала)
- Выбор из набора готовых обложек (12 шт, иллюстрации/паттерны)
- Обложка отображается на главном экране при открытии месяца
- Рисование своей обложки (canvas) — v1.1

#### 3.2 Календарь месяца
- Классическая сетка месяца (пн–вс)
- Цветовая индикация настроения на каждом дне
- Индикаторы: события, задачи, срок годности косметики
- При тапе на день — детальный вид дня

#### 3.3 Задачи месяца
- Список задач для текущего месяца
- Поля: название (обязательно), дата (опционально), статус (выполнена / не выполнена / отложена), связанное событие (опционально)
- Задачи без даты — в разделе "Без даты"
- Задачи с датой — отображаются в соответствующем дне календаря

#### 3.4 События
- Привязаны к конкретной дате + время
- Поля: название, время, место (опционально), описание
- Можно привязать к задаче (или оставить самостоятельным)
- Цветовая метка (5–6 цветов)

#### 3.5 Трекер косметики
- Сканирование штрихкода камерой (ML Kit, on-device)
- По штрихкоду: поиск в Open Beauty Facts → ручной ввод если не найдено
- Поля: название, бренд, тип продукта, дата открытия (по умолчанию сегодня), PAO
- PAO: ручной ввод или дефолт по типу (тушь 3-6M, тональный 6-12M, помада 12-18M, тени 24M, духи 36M+)
- Дата истечения = дата открытия + PAO (автоматически)
- Автоматическое событие в календаре на дату истечения
- Список косметики: активная / истекает скоро / просрочена
- **Важно:** Open Beauty Facts слабо покрывает российский рынок — ручной ввод будет основным путём

#### 3.6 Mood tracker
- 5 иконок настроения: отлично / хорошо / нейтрально / плохо / ужасно
- Одна отметка в день (можно менять до конца дня)
- Цвет дня в календаре определяется настроением
- Year in Pixels — сетка 365 дней с цветами настроения

#### 3.7 Streak
- Счётчик непрерывных дней заполнения (mood + хотя бы 1 действие)
- Отображается на главном экране
- Milestone-бейджи: 7, 30, 100, 365 дней

#### 3.8 Бэкап (JSON-экспорт + авто-синк) ✅
- JSON-экспорт: скачать полный снапшот всех данных (кнопка на главной)
- Авто-синк в облако (n8n): при каждом изменении данных, debounce 3 сек
- Восстановление из облака: кнопка на главной странице
- Периодический автоматический бэкап — не нужен (авто-синк покрывает потребность)
- Риск потери данных при смене устройства/браузера закрыт

### MVP-1: Расширение (после feedback от Кати)

#### 3.9 Стикеры
- Базовый набор 50-100 шт (botanical / watercolor стиль)
- Размещение на обложках и страницах календаря
- Drag & drop, масштабирование, поворот

#### 3.10 Migration
- Мягкий вариант: напоминание в конце месяца, БЕЗ блокировки
- Пошаговый обзор незавершённых задач (по одной)
- Для каждой: "Перенести в новый месяц" / "Отменить" / "В Future Log"
- Нельзя "перенести всё" одной кнопкой
- Можно пропустить весь обзор

#### 3.11 Future Log
- Обзор на 6 месяцев вперёд (горизонтальный скролл)
- Парковка задач и событий на конкретный будущий месяц
- Задачи автоматически появляются в Monthly Log когда месяц наступает

#### 3.12 Collections
- Тематические страницы вне календаря
- Пресеты: Books to Read, Wishlist, Gift Ideas, Brain Dump
- Кастомные коллекции (название + иконка)
- Элементы: текст, checkbox, дата (опционально)

## 5. Хранение данных

- Всё локально на устройстве (без сервера, без облака)
- Формат: SQLite
- Бэкап: JSON-экспорт (уже в MVP-0)

**Таблицы MVP-0:**
```
months         — id, year, month, cover_type, cover_data
tasks          — id, month_id, title, date?, status, event_id?
events         — id, date, time?, title, description?, place?, color, task_id?
cosmetics      — id, barcode?, name, brand?, type?, photo_url?, open_date, pao_months, expire_date, event_id?
covers         — id, asset_name (встроенные обложки)
moods          — id, date, level (1-5), note?
streak_data    — id, current_streak, longest_streak, last_active_date
```

**Добавятся в MVP-1:**
```
sticker_placements — id, page_type, page_id, sticker_id, x, y, scale, rotation
collections        — id, title, type, icon?, created_at
collection_items   — id, collection_id, text, checked?, date?, sort_order
future_log         — id, month, year, title, type (task/event), migrated?
```

## 6. Стек технологий

**Решено: Vanilla JS (PWA)**

| | |
|---|---|
| **Платформа** | Web (PWA — Progressive Web App) |
| **Стек** | Vanilla HTML/CSS/JS, без фреймворков |
| **Хранение** | IndexedDB (локально в браузере) |
| **Установка** | "Добавить на главный экран" на iOS/Android |
| **Сканер** | `html5-qrcode` или `quagga2` (камера через браузер) |
| **Offline** | Service Worker для работы без интернета |

**Почему PWA:** работает на любом устройстве, не нужен App Store, быстрый старт разработки, Катя просто открывает URL и добавляет на экран.

**Ограничения PWA:** push-уведомления на iOS ограничены (с iOS 16.4 работают, но хуже чем нативные), камера работает через getUserMedia (не все функции ML Kit доступны).

**API для косметики:**
- Open Beauty Facts: `world.openbeautyfacts.org/api/v2/product/{barcode}.json` (бесплатно, 100K+ продуктов)
- Fallback: ручной ввод

**PAO дефолты** встроены в приложение по типу продукта.

## 8. Дизайн-направление

- Стиль: **"Уютный дневник"** (cozy / cottagecore / warm botanical)
  - Тёплые пастельные тона, мягкие текстуры, ощущение бумажного журнала
  - Натуральные оттенки: крем, беж, приглушённый розовый, шалфей, лаванда
  - Скруглённые углы, мягкие тени, "бумажная" текстура фона
- Типографика: serif-шрифт для заголовков (Cormorant Garamond), sans-serif для контента (Inter)
- Тема: светлая тёплая (основная), тёмная (v2)

### Идеи от Кати (v2+)

1. **Трекер лекарств** — аналог трекера косметики, но для медикаментов. Срок годности, дата открытия, напоминания о приёме/замене. Та же механика: сканер штрихкода → PAO → событие в календаре.

2. **Парсинг скриншотов** — заскринил переписку где договорились о встрече → приложение извлекает дату, время, место и создаёт событие в календаре автоматически. (OCR + AI для извлечения структурированных данных из текста)

3. **Внутренний ребёнок** — вместо виртуального питомца: загружаешь свою детскую фотографию → приложение создаёт мультяшную версию тебя-ребёнка → ухаживаешь за ним (кормишь, играешь, укладываешь спать). Мотивация заботиться о себе через метафору заботы о внутреннем ребёнке.

## 13. Критические риски

1. **Open Beauty Facts** — мало российской косметики. Сканер может разочаровать (4 из 5 не найдутся). Mitigation: ручной ввод как основной путь, сканер как бонус
2. **Обложки** — нужно 12+ иллюстраций, источник не определён. Блокирующий вопрос для MVP-0
3. **Стек не выбран** — влияет на timeline и возможности. Нужно решить до начала разработки
4. **Масштабирование** — если захотим на рынок, нужны: auth, облачный бэкап, App Store review, монетизация
